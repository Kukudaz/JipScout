# 현재 우선 작업

## 1순위
기존 예산 계산기 구조를 대출 가능액 판정기 구조로 바꾼다.

## 2순위
입력 항목을 최소화한다.
삭제 또는 숨김 대상:
- 평수
- 방 수
- 화장실 수
- 방향
- 현관 방향
- 역거리
- 구조 선호
- 연식
- 수리 가능 여부
- 예비비
- 이사비
- 가전/가구비
- 수리비
- 결혼비용

## 3순위
추가할 입력:
- 혼인 상태
- 직업 형태
- 나이
- 지역 구분
- 체증식 관심 여부

## 4순위
결과 카드 우선순위를 바꾼다.
새 우선순위:
1. 예상 최종 대출 가능액
2. 디딤돌 예상액
3. 보금자리론 예상액
4. 일반 주담대 예상액
5. 체증식 가능 여부
6. 총 구매 가능 금액

## 5순위
계산 로직을 상품별로 나눈다.
- 디딤돌 판정 함수
- 보금자리론 판정 함수
- 일반 주담대 판정 함수
- 체증식 판정 함수
- 최종 통합 결과 함수

## 6순위
완료 후 typecheck와 build를 확인한다.
