# 프로젝트 목적
이 프로젝트는 한국 아파트 실거주 사용자를 위한 대출 가능액 판정 웹 MVP를 만든다.

# 현재 단계
1단계 MVP만 구현한다.
현재 단계의 핵심은 사용자가 몇 가지 정보만 입력하면
정부대출 가능성, 일반 주담대 가능성, 체증식 가능성, 예상 대출 가능 금액 범위를 빠르게 보여주는 것이다.

# 제품 방향
- 이 서비스의 첫 목적은 "어떤 집이 좋냐"가 아니라 "내가 대출이 얼마까지 나오냐"를 빠르게 알려주는 것이다.
- 예산 계산기보다 대출 판정기가 먼저다.
- 주택 세부 조건 추천은 2단계 이후에 다룬다.
- 1단계에서는 입력 부담을 최소화한다.

# 기술 스택
- Next.js
- TypeScript
- Tailwind CSS

# 작업 원칙
- 한 번에 기능 하나씩만 구현한다.
- 먼저 계획을 짧게 제시한 뒤 구현한다.
- 계산 로직은 UI와 분리한다.
- 타입은 별도 파일로 분리한다.
- 하드코딩은 최소화한다.
- 수정 범위는 필요한 파일만 최소로 유지한다.
- 작업 후 타입 에러와 빌드 에러를 우선 확인한다.
- 기존 코드가 있어도 현재 단계 목표에 맞지 않으면 과감히 단순화한다.

# 1단계 입력 원칙
1단계에서는 아래 입력만 중요하다.
- 본인 연소득
- 배우자 연소득
- 보유 현금
- 기존 월 부채 상환액
- 무주택 여부
- 생애최초 여부
- 혼인 상태
- 직업 형태
- 나이
- 희망 주택 가격
- 지역 구분
- 체증식 관심 여부

# 1단계에서 제외할 입력
아래 항목은 1단계 기본 화면에서 제외한다.
- 평수
- 방 수
- 화장실 수
- 방향
- 현관 방향
- 역거리
- 구조 선호
- 연식
- 수리 가능 여부
- 예비비
- 이사비
- 가전/가구비
- 수리비
- 결혼비용

# UI 원칙
- 한국어 UI로 만든다.
- 첫 화면 입력은 최대한 적고 단순하게 만든다.
- 결과는 숫자 중심으로 크게 보여준다.
- 가장 먼저 보여줄 것은 "예상 최종 대출 가능액"이다.
- 그 다음으로 정부대출 가능성, 일반 주담대 가능성, 체증식 가능성을 보여준다.
- 모바일에서도 크게 깨지지 않게 만든다.
- 디자인보다 사용성과 가독성을 우선한다.

# 1단계 핵심 출력
- 예상 최종 대출 가능액
- 디딤돌 예상 가능 여부와 예상액
- 보금자리론 예상 가능 여부와 예상액
- 일반 주담대 예상 가능액
- 체증식 가능 여부
- 보유 현금 포함 총 구매 가능 금액
- 간단한 판단 이유

# 도메인 원칙
- 결과는 확정이 아니라 예상치로 보여준다.
- 실제 대출 가능 여부는 금융기관 심사와 최신 규제에 따라 달라질 수 있다는 문구를 반드시 포함한다.
- 사용자에게 "가능/불가"만 보여주지 말고 가능한 경우 예상 금액 범위를 함께 보여준다.
- 정부대출은 "가능성 있음 / 조건부 / 어려움"처럼 설명형으로 보여준다.
- 체증식은 별도 항목으로 분리해 보여준다.

# 파일 구조 원칙
- 계산 로직: lib/
- 타입 정의: types/
- UI 컴포넌트: components/
- 페이지: app/

# 코드 스타일
- 함수명은 명확하게 작성한다.
- 숫자 단위는 일관되게 유지한다.
- 현재 프로젝트는 금액 단위를 "만원" 기준으로 사용한다.
- 불필요한 라이브러리는 추가하지 않는다.
- 주석은 필요한 곳에만 짧게 작성한다.

# 요청 방식
- 내가 큰 요구를 하더라도 한 번에 전부 만들지 말고,
  현재 단계에서 필요한 최소 기능부터 만든다.
- 애매하면 과도하게 추측하지 말고 보수적으로 구현한다.
- 먼저 동작하는 MVP를 만들고 이후 개선한다.
